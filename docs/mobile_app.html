Design and Progress

Mobile Application

WaiterBot mobile application is for placing the orders by the customers of the restaurent thereby replacing the traditional menu card ordering system. When a customer visits a restaurent, the customer can scan the QR code on the table by using the mobile application. The QR code will contain the shop identification number and the table identification number. Once the QR code scanning is complete, the customer can place the order.

When placing the orders, customers can get an idea of the food item by the information provided such as images, potion sizes, reviews by other customers, ingredients, etc. This information will help the cusotmer to have a good experience at the restaurent. Unlike the traditional menu card ordering system, information on the application will be up to date.

Once the order is place, the customer will be notified with the progress of the preparation and the delivery process. When the food items are delivered, the customer can post a reviews if the customer is a registered user.

UI/UX and Other Features

  • Small app size ~ 18mb
  • Self describling interface
  • Seamless navigation without lagging
  • Guest login feature for non-registered customers
  • Real time notifications
  • Cashless payments via secure gateway

Flutter

Since we have to target the most popular mobile operating systems like iOS and Android we developed our mobile application using Flutter. With Flutter we can build cross platform mobile applications with single codebase.
